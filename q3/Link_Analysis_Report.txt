Link Analysis Report
====================

1) Create original file
Command:
echo "Linux link test file" > original.txt
Output:
(no output)
Explanation:
This created the source file that would be used for the link experiment.

2) Create hard link and symbolic link
Command:
ln original.txt hardlink.txt
ln -s original.txt symlink.txt
Output:
(no output)
Explanation:
A hard link points to the same inode as the original file, while a symbolic link stores a path to the target.

3) Inode and metadata information
Command:
ls -li
stat original.txt hardlink.txt symlink.txt
Output:
1310805 -rw-rw-rw- 2 codespace codespace 21 Jul 3 13:54 hardlink.txt
1310805 -rw-rw-rw- 2 codespace codespace 21 Jul 3 13:54 original.txt
1310806 lrwxrwxrwx 1 codespace codespace 12 Jul 3 13:54 symlink.txt -> original.txt
File: original.txt
Size: 21 Blocks: 8 IO Block: 4096 regular file
Device: 7,4 Inode: 1310805 Links: 2
...
Explanation:
The hard link shares the same inode as the original file, while the symbolic link has a different inode and points to the original path.

4) Read contents through each link
Command:
cat original.txt
cat hardlink.txt
cat symlink.txt
Output:
Linux link test file
Linux link test file
Linux link test file
Explanation:
All three names accessed the same content, showing that both link types reference the same data initially.

5) Delete original file
Command:
rm original.txt
Output:
(no output)
Explanation:
Deleting the original name tests whether the content remains accessible through the other link.

6) Observe behavior after deletion
Command:
ls -li
cat hardlink.txt
cat symlink.txt
Output:
1310805 -rw-rw-rw- 1 codespace codespace 21 Jul 3 13:54 hardlink.txt
1310806 lrwxrwxrwx 1 codespace codespace 12 Jul 3 13:54 symlink.txt -> original.txt
Linux link test file
cat: symlink.txt: No such file or directory
Explanation:
The hard link still worked because it points to the same inode, but the symbolic link broke because it still references the deleted original path.

Conclusion:
Hard links are directory entries to the same inode and survive deletion of the original name as long as the inode remains. Symbolic links are separate entries that reference a path and break when the target is removed.
